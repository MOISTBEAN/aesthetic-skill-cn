# Chinese Product Structure Map

Map product category to operational structure before selecting visual styling.

## Finance dashboard

**Use:** thesis tracker, risk trigger panel, decision log, compact comparison table, valuation range, catalyst tracker, next-review action.

**Primary flow:** identify exception → inspect original thesis and sell condition → compare latest evidence → record decision → set next review.

**Avoid:** crypto neon, all-green states, fake charts, generic KPI overview, unexplained scores, AI buy recommendations.

## Xiaohongshu creator tool

**Use:** content calendar, topic library, note status, cover and title review, brand CRM, approval queue, mobile preview.

**Primary flow:** select topic → prepare assets → review cover and title → approve → schedule → review result.

**Avoid:** full pink UI, `一键爆款`, fake engagement predictions, generic content cards, platform-brand imitation.

## AI review SaaS

**Use:** input / output split, evidence panel, version difference, human review queue, risk label, approval action, audit history.

**Primary flow:** open source → inspect finding and evidence → accept, reject, or request input → assign reviewer → preserve audit record.

**Avoid:** generic AI productivity hero, `AI-powered everything`, chat box as the whole product, fake accuracy metrics, decorative terminal previews.

## Mapping rule

The chosen structure must make at least three modules impossible to reuse unchanged in another category. If it does not, return to [PRODUCT_REALITY_AUDIT.md](PRODUCT_REALITY_AUDIT.md).
