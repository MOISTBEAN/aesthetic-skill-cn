# Real Content Library

Use this as a module selection library, not a checklist to paste wholesale. Select modules that form a complete decision path. Every selected module must expose an object, state, and action.

## Finance research

| Module | Required content | User action |
|---|---|---|
| 买入理由 | original thesis, evidence date, intended holding period | 复核原始假设 |
| 原定卖出条件 | condition, threshold, current state | 确认是否触发 |
| Thesis 状态 | unchanged / partially broken / broken with latest evidence | 更新判断 |
| 风险触发项 | trigger, severity, evidence, owner | 标记已核对 |
| 估值区间 | method, range, current price, update date | 比较当前估值 |
| 催化剂跟踪 | event, expected date, changed date, result | 调整复核日期 |
| 回撤原因拆解 | market, industry, company factors | 记录主要原因 |
| 交易复盘 | trade, original reason, rule adherence, lesson | 补充复盘 |
| 持仓纪律评分 | named rules and violations, not a magic score | 打开违规项 |
| 观察列表 | thesis state, valuation, catalyst, risk | 进入研究详情 |
| 待核对事项 | exception, deadline, consequence | 立即复核 |
| 下次复核日期 | trigger or calendar date and owner | 安排复核 |

## Selection rule

Choose enough modules to answer: what changed, why it matters, what state it is in, what the user decides, and what happens next. Remove modules that only make the page look busy.
