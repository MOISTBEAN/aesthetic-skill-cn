# Rejected: Over-Polished Workflow-Empty Dashboard

## Candidate

A balanced dashboard uses warm neutrals, soft shadows, twelve rounded cards, four large metrics, a decorative trend chart, and a polished side panel. Labels are concise, but users cannot open an exception, inspect a source, resolve a failure, or record a decision.

## Why it fails

- polish replaces workflow
- metrics have no source or supported decision
- equal cards erase priority
- negative states are decorative labels without recovery actions
- the layout could represent any product after a name change

## Violated controls

- Output Contract
- Visual Primitive Lock
- Product Swap Test
- Fake Dashboard Test
- Workflow Reality Requirement
- Negative State Requirement

## Required rewrite

Select a generic page type and primitive set first. Remove every metric, chart, and card that cannot name its source, state, user action, and next step. Rebuild around the primary queue, object, exception, or form flow.
