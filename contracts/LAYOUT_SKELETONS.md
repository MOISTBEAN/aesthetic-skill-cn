# Layout Skeletons

Free layout generation is not allowed. Select one skeleton and preserve its hierarchy across desktop and mobile.

If the output does not match a skeleton, it fails.

## 1. Workbench Layout

**Use for:** Xiaohongshu Creator Workbench or AI Review tools.

**Desktop structure:**

1. left sidebar or compact top navigation
2. main workflow area
3. right detail or preview panel
4. bottom or secondary queue
5. visible semantic status tags

**Mobile order:** context → urgent queue → selected workflow → detail / preview → secondary queue.

Do not turn the main workflow into a dashboard card grid.

## 2. Review Queue Layout

**Use for:** AI Review / Human Approval Workflow.

**Desktop structure:**

1. top status summary with actionable exceptions
2. left review queue
3. center selected input and AI result
4. right evidence, risk, and decision panel
5. sticky action bar for 退回修改 / 标记风险 / 确认通过

**Mobile order:** selected queue item → difference → evidence → decision action → remaining queue.

The selected item must remain visually dominant. Summary metrics may not replace the queue.

## 3. Calendar + Detail Layout

**Use for:** Xiaohongshu Creator Workbench.

**Desktop structure:**

1. top account and status filters
2. calendar or publishing schedule grid
3. content card list tied to calendar selection
4. mobile preview panel
5. revision queue

**Mobile order:** filters → selected day → content list → revision queue → preview.

Each calendar item must show owner, state, and next action. Do not show only thumbnails.

## 4. Local Landing Layout

**Use for:** service businesses.

**Structure:**

1. compact hero naming concrete service, area, and booking action
2. service cards with scope and price range
3. process steps with customer preparation
4. pricing and booking area with availability state
5. location,营业时间, transport, phone, and WeChat contact block

The hero must not exceed 30% of the first desktop viewport. Do not add dashboard sections.

## 5. Finance Workspace Layout

**Status:** WIP only.

**Desktop structure:**

1. compact top risk summary
2. thesis tracker table
3. selected holding detail
4. risk trigger panel
5. decision log and next review action

Do not promote this skeleton as public proof until human review approves the result.

## Skeleton audit

Before final delivery, map every major page region to the numbered skeleton. Missing, reordered, or additional regions require an explicit product reason. Otherwise the output fails.
